记录下这几天的工作

  • Markdown
  • git
  • 随笔

自从上次一时兴起学别人搞了个博客，注册了域名，又注册了github之后，玩了几天，就丢那里了。原因嘛，无非一懒二懒三还是懒…这次又学了学怎么装虚拟机，在虚拟机里安装Ubuntu系统，又把博客捡起来了。这里就随意记录下这几天干了什么吧。

t440s是win8.1的系统，首先是机器很好，用着很顺心。装了VMware workstation pro的虚拟机软件，又下了Ubuntu的系统，实验室的有线网还是很快的，可能是系统镜像在教育网里的原因。在vm里装了Ubuntu系统后，想起了室友说vm有unity模式，就是可以在主机和虚拟机间直接进行文件操作。我想我可以把学习开发工作放在Ubuntu虚拟机里，文字工作需要点好看的界面和便利的操作，就放在主机的win8.1里。所以这个unity就很重要了，比如可以把我写的东西，比如现在这篇直接复制到Ubuntu里，然后在linux下进行hexo博客的维护。我计划是把这些文章和主题配置什么的推送到github里存起来，这样换电脑，这些东西到不至于丢。

markdown的语法是简单，但是总要查查，这里附个说明。

如何推送github，我也学习了一下，主要参考了这篇博客的内容，讲如何添加文件到仓库。另外还有怎么推送，参考这篇博客。
主要代码就这么几条

git add 文件名或者文件夹名

git commit -m "注释（就是这些加的文件是干了什么）"

git remote add origin git@github.com:用户名/远程仓库.git

git push origin master
